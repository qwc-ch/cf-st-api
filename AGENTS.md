# SillyTavern Backend — Agent Guide

## Stack

- SvelteKit 2 + Svelte 5 (server-only API + static frontend)
- Adapter: `@sveltejs/adapter-vercel` — Serverless functions on Vercel
- TypeScript strict, moduleResolution `bundler`
- Database: PostgreSQL (Neon Serverless via `@neondatabase/serverless`)
- File storage: Local filesystem (`./data/storage/`) by default; S3-compatible (`STORAGE_BACKEND=s3`) for production. Supports R2, AWS S3, MinIO. Vercel Blob can be swapped in `src/lib/r2.ts`.

## Commands

```bash
pnpm dev              # vite dev
pnpm build            # vite build
pnpm preview          # vite preview
pnpm lint             # biome lint src/
pnpm format           # biome format --write src/
pnpm check            # biome check --write src/ (format + lint + organize imports)
```

There is no `typecheck` or `test` script. No test framework is configured.

## Database

- PostgreSQL via `@neondatabase/serverless` (`src/lib/db.ts`).
- Schema SQL in `migrations/0001_initial.sql` — run in Neon Console or Supabase Dashboard.
- Tables: `users`, `characters`, `chats`, `messages`, `settings`, `world_infos`, `chat_groups`, `backgrounds`, `tags`, `settings_snapshots`, `presets`, `secrets`.
- Database connection string via `DATABASE_URL` environment variable.
- Queries use positional params (`$1, $2, ...`) via the Neon `sql` tagged template / function.

## Auth

- Cookie-based sessions (HMAC-signed, no JWTs). Cookie name: `sillytavern_session`.
- `SESSION_SECRET` can be set as an environment variable.
- Session expiration: 24h, encoded in the token payload.
- `event.locals.user` is set by `hooks.server.ts` on every request.
- `requireAuth(event)` throws `Error('Unauthorized')` if no valid session.

## API structure

All routes under `src/routes/api/`. Each subdirectory is a SvelteKit server route (`+server.ts`). The API is REST-style:
- `/api/users` — user CRUD
- `/api/characters` — character CRUD
- `/api/chats` — chat & message CRUD
- `/api/groups`, `/api/worldinfo`, `/api/backgrounds`, `/api/tags`, `/api/stats`
- `/api/settings` — user settings + snapshot management
- `/api/presets` — preset save/delete/restore
- `/api/secrets` — API key management
- `/api/files`, `/api/images` — file upload/download (local filesystem)
- `/api/backends/chat-completions`, `/api/backends/text-completions` — generic LLM proxies
- `/api/backends/kobold` — KoboldAI generate/status/transcribe/embed
- `/api/openai`, `/api/anthropic`, `/api/google`, `/api/novelai` — provider-specific endpoints
- `/api/horde` — AI Horde text/image generation
- `/api/sd` — Stable Diffusion WebUI + ComfyUI + 15+ image providers
- `/api/openrouter`, `/api/nanogpt` — model/provider listings
- `/api/azure`, `/api/volcengine`, `/api/minimax` — additional LLM backends
- `/api/speech`, `/api/translate` — TTS/STT and translation
- `/api/vector` — text embeddings
- `/api/themes`, `/api/avatars`, `/api/sprites`, `/api/moving-ui` — UI resources
- `/api/extensions` — extension install/update/delete/move/switch/discover/version/branches/execute (git-based, stored in `./data/extensions/`)
- `/api/quick-replies` — extensions
- `/api/content` — URL content import
- `/api/backups`, `/api/data-maid`, `/api/image-metadata` — data tools
- `/api/tokenizers` — token counting (stub)
- `/api/ping` — health check
- `/api/plugins` — (empty, reserved)
- `/csrf-token` — CSRF token endpoint
- `/thumbnail` — image thumbnail redirect

## File storage

- Local filesystem at `./data/storage/` (configurable via `STORAGE_DIR` env var).
- Uploads stored at `{userHandle}/{category}/{filename}`.
- Allowed MIME types: images (png/jpeg/webp/gif/bmp), plus avif, video (mp4/webm), audio (mpeg/ogg/wav/flac/aac/mp4).
- `getPublicUrl()` returns `PUBLIC_STORAGE_URL` prefixed URL if set, otherwise falls back to `/api/files/raw/{key}`.
- For production, replace with Vercel Blob or S3-compatible storage.

## Environment Variables

| Variable | Description |
|---|---|
| `DATABASE_URL` | PostgreSQL connection string (Neon) |
| `SESSION_SECRET` | HMAC signing key for session cookies |
| `PUBLIC_STORAGE_URL` | Public URL prefix for uploaded files |
| `STORAGE_DIR` | Local filesystem storage directory (default: `./data/storage`) |
| `GITHUB_CLIENT_ID` | GitHub OAuth client ID |
| `GITHUB_CLIENT_SECRET` | GitHub OAuth client secret |
| `ALLOWED_GITHUB_USERS` | Comma-separated list of allowed GitHub usernames |
| `FRONTEND_URL` | Frontend URL for GitHub OAuth redirect |
| `ADMIN_USERNAME` | Admin login username (simple auth) |
| `ADMIN_PASSWORD` | Admin login password (simple auth) |
| `EXTENSIONS_DIR` | Extensions storage directory (default: `./data/extensions`) |

## Notable quirks

- No test infrastructure exists.
- Formatter/linter: Biome — config in `biome.json` (4-space indent, single quotes, 120 width). Use `pnpm check` to auto-format all files.
- `tsconfig.json` extends `.svelte-kit/tsconfig.json` (generated by SvelteKit on `pnpm dev` or `pnpm build`).
- `src/hooks/` directory exists but is empty — not used.
- Frontend static files in `static/` are served automatically by Vercel.
- S3 storage: set `STORAGE_BACKEND=s3` + `S3_ACCESS_KEY_ID`, `S3_SECRET_ACCESS_KEY`, `S3_BUCKET`, optionally `S3_ENDPOINT` (for R2/MinIO), `S3_REGION`, `S3_FORCE_PATH_STYLE`. Requires `pnpm add @aws-sdk/client-s3`.
